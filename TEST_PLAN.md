# План Полного Тестирования Карточки (Full Field Testing Plan 2026)

Этот регламент описывает процедуру проверки каждого значения, вносимого автоматизацией. Тестирование считается успешным только при подтверждении записи данных в БД Яндекса (через Read-Back).

## 1. Раздел: Данные и Контакты (Basic Info)
| Поле | Тип теста | Ожидаемый результат |
| :--- | :--- | :--- |
| Обычное название | Edit & Read-Back | DOM содержит строку "Клубика" |
| Короткое название | Length Check (max 25) | Обрезается до 25 или выдает Alert |
| Описание | LSI Check | Наличие ключей "SEO", "маркетинг" |
| Email | Persistence | Поле не пустое после Reload |
| Сайт | Link Validation | URL начинается с https:// |
| Соцсети | "Add" Logic | Вторая ссылка добавляется без затирания первой |

## 2. Раздел: График (Schedule)
| Сценарий | Метод проверки | Статус успеха |
| :--- | :--- | :--- |
| Режим 24/7 | Click Button | Флаг "Круглосуточно" активен |
| Перерывы | Input Range | Наличие интервала "12:00-13:00" |
| Праздники | Future Date Set | Появление плашки "Работает иначе" |

## 3. Раздел: Медиа (Media & Geo)
| Ассет | Проверка EXIF | Проверка Категории |
| :--- | :--- | :--- |
| Фото входа | GPS Match (56.29, 43.99) | Находится в теге `enter` |
| Интерьер | Visual Contrast | Текст читаем на 20% яркости |
| Логотип | Aspect Ratio | Соотношение сторон 1:1 |
| Видео | Metadata Check | Формат MP4/MOV подтвержден |

## 4. Раздел: Товары и Услуги (Catalog)
| Компонент | Проверка |
| :--- | :--- |
| YML Feed | Валидация XML-схемы |
| Цены | Числовой формат (RUR) |
| SEO-описание услуг | Плотность ключевых слов > 3% |

## 5. Раздел: Лидогенерация (Promos/Stories)
- **Акции**: Проверка наличия кнопки "Опубликовать".
- **Сториз**: Проверка интерактивной кнопки (URL-валидация).
- **События**: Проверка дат начала и окончания.

## 6. Алгоритм Автоматического Теста (The Audit Script)
Скрипт `scripts/final-report.ts` должен запускаться в конце каждого цикла. 
**Критерии прохождения (Gatekeepers):**
1. Отсутствие ошибок `TimeoutError` в логах.
2. Скриншот `data/FINAL_PROOF.png` визуально чист (нет поп-апов).
3. `ADVICE.md` содержит не более 2 предупреждений критического уровня.

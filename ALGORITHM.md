# Универсальный Алгоритм Деконструкции Бизнеса (Business Deconstruction Algorithm 2026)

Этот документ описывает методологию, по которой наши ИИ-агенты разбирают любой бизнес «до молекул» для создания идеальной карточки в Яндекс Бизнесе.

## 1. Новая Персона: Аналитик Ниши (Niche Analyst)
Его задача — до начала любой автоматизации провести рентген бизнеса по 4 осям:

### А. Молекула Семантики (SEO-Ядро)
- **Алгоритм**: Парсинг топ-10 конкурентов в Картах по целевому региону.
- **Действие**: Извлечение не только прямых ключей (напр. "Стоматология"), но и скрытых LSI-запросов ("лечение без боли", "микроскоп", "гарантия на импланты").
- **Результат**: Список из 50+ опорных слов для SEO-Специалиста.

### Б. Молекула Доверия (Media-Стандарты)
- **Алгоритм**: Определение "критических триггеров" ниши.
- **Примеры**:
  - *Общепит*: Фокус на еде (Food-porn) + чистота кухни (Interior).
  - *Услуги/B2B*: Лица экспертов + сертификаты + процесс работы.
  - *Медицина*: Стерильность + оборудование + лицензии.
- **Результат**: ТЗ для Медиа-Дизайнера по композиции кадров и вшиваемым мета-данным.

### В. Молекула Конверсии (User Intent)
- **Алгоритм**: Анализ того, ЗАЧЕМ пользователь открыл карточку.
- **Триггеры**:
  - "Хочу сейчас" (Кнопка: Позвонить/Проложить маршрут).
  - "Выбираю лучшее" (Кнопка: Посмотреть отзывы/Прайс).
  - "Ищу выгоду" (Кнопка: Получить акцию).
- **Результат**: Настройка кнопок в Сторис и Акциях Контент-Менеджером.

## 2. Пошаговый Процесс Масштабирования (Scalling Algorithm)

### Шаг 1: Инициация (Run Once)
Аналитик Ниши получает вводные (Ниша + Регион). Генерирует файл `niche_config.json`.

### Шаг 2: Настройка Фундамента (Architecture)
1. **SEO-Специалист**: Заполняет рубрики и описание на основе `niche_config`.
2. **Медиа-Дизайнер**: Генерирует обложки и вшивает GPS по молекуле Доверия.

### Шаг 3: Цикл Активности (Maintenance)
1. **Контент-Менеджер**: Генерирует посты, используя психологические триггеры ниши (напр. "Страх опоздать" для акций или "Социальное доказательство" для отзывов).
2. **Reputation Менеджер**: Отвечает на отзывы в тональности бренда (Tone of Voice), определенной Аналитиком (напр. "Дружелюбный" для кафе или "Экспертный" для юристов).

## 4. Почему это Эталон?
Мы используем **Байесовский анализ вероятности выбора**: алгоритм рассчитывает, какой набор данных в карточке с наибольшей вероятностью приведет к клику в данной конкретной категории. Это не просто "заполнение полей", это математическая модель захвата внимания.
